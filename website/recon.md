# @lukekaalim/act-recon

Reconciler implementation for @lukekaalim/act.

## What is a Reconciler

A reconciler's broad job is to manage the state of a tree,
and when new changes are proposed to the tree, figure
out what needs to be added/removed/updated.

It works closely with a Renderer, sending it's output
directly to the renderer.

A reconciler is more specifically made up of a few sub systems:
  - [Scheduler](#scheduler)
  - [CommitTree](#commit-tree)
  - [WorkThread](#work-thread)
  - [ElementService](#element-service)
